## 🧭 上篇：CTF Web 学习路线与避坑指南

很多新手容易犯的第一个错误，就是连 HTTP 协议都没搞懂，就直接去背 SQL 注入的 Payload。基础不牢，地动山摇。

### 第一阶段：筑基期（绝对不要跳过！）
*   **🌐 网络基础（重中之重）：** 深刻理解 HTTP/HTTPS 协议。请求头 (Headers)、响应头、Cookie、Session、状态码 (200, 302, 403, 500 等)。
*   **🖥️ 前端基础：** HTML/CSS（看懂表单提交），JavaScript（DOM 操作，XSS 的基础），浏览器安全策略（同源策略 SOP、跨域 CORS）。
*   **⚙️ 后端与底层环境：** Linux 基础命令（文件操作、权限、管道符）；MySQL 基础语法；至少掌握一门后端语言（首推 PHP，其次 Python），能看懂代码逻辑即可。

### 第二阶段：核心漏洞图谱（经典漏洞实战）
掌握 Web 安全中最经典的漏洞模型。不仅要知道“怎么黑”，更要知道“为什么能黑”（成因）以及“怎么修”（防御）。
*   **注入类漏洞：** SQL 注入、命令注入 (RCE)。
*   **文件操作类：** 文件上传（绕过黑白名单）、文件包含 (LFI/RFI)。
*   **客户端漏洞：** XSS (跨站脚本攻击)、CSRF。
*   **逻辑漏洞：** 越权访问、支付逻辑漏洞、验证码绕过。

### 第三阶段：进阶与现代 Web 安全（拉开差距的考点）
*   **反序列化漏洞：** PHP 反序列化 (POP 链)、Java 反序列化 (CC 链)、Python Pickle。
*   **服务端请求伪造：** SSRF (内网探测、打 Redis/MySQL)。
*   **模板注入 (SSTI)：** Jinja2、Twig 等引擎的沙箱逃逸。
*   **现代特性：** Node.js 原型链污染、JWT 伪造。

### 🛑 防走弯路指南（避坑必看）
1.  **🚫 戒掉对自动化工具的依赖：** 新手期绝对不要一开始就用 Sqlmap 或 AWVS。如果连手动注入的闭合都没搞懂，扫描器只会让你知其然而不知其所以然。
2.  **🔍 看重源码，学会本地调试：** 遇到黑盒题目做不出，试着在本地写一段类似的 PHP 代码，`var_dump()` 打印一下变量，看看底层到底发生了什么。
3.  **📖 赛后复盘重于比赛本身：** 比赛时做不出很正常。比赛结束后，一定要去看别人的 Writeup！学习大佬的切入点和 Bypass 思路。
4.  **🧱 抓核心，轻偏门：** 很多 CTF 题目的 Trick 非常偏门，不要花太多时间死记硬背偏门函数，优先掌握底层的核心机制。

## 🗺️ 下篇：CTF Web 核心知识体系字典

本字典旨在对 Web 安全领域的关键技术进行系统性分类，作为日常学习、做题与代码审计的速查索引。

### 一、 信息搜集与敏感泄露 (Information Gathering)
在 CTF 中，找不到源码往往连题目的门都进不去。
*   **源码泄露：** `.git` 泄露、`.svn` 泄露、`.DS_Store` 泄露。
*   **备份文件：** 常见的备份后缀如 `.bak`, `.zip`, `.tar.gz`, `~`。
*   **编辑器缓存：** Vim 缓存泄漏 (`.swp`)。
*   **探针与配置泄露：** `phpinfo()` 页面、`robots.txt`、Spring Boot Actuator 接口泄露。

### 二、 协议与网络基础层
*   **HTTP 请求结构：** Method, URI, Version, Headers (Host, User-Agent, X-Forwarded-For)。
*   **状态保持机制：** Cookie (HttpOnly, Secure 属性), Session, JWT 签名机制。
*   **同源策略与跨域：** SOP 限制范围, CORS 配置不当, JSONP 劫持。

### 三、 核心漏洞图谱 (服务端)
**1. 注入类漏洞 (Injection)**
*   **SQL 注入：** 联合查询、报错注入、布尔盲注、时间盲注、堆叠查询。
    *   *进阶技巧：* 绕过空格/逗号/单引号、无列名注入、DNSLog 外带数据。
*   **命令注入 (RCE)：** 管道符利用 (`|`, `&`, `;`)、无回显 RCE。

**2. 文件操作漏洞**
*   **文件包含 (LFI / RFI)：** 伪协议利用 (`php://filter`, `data://`)、日志包含 (Log Poisoning)、Session 包含。
*   **文件上传 (File Upload)：** MIME 绕过、后缀绕过、.htaccess/.user.ini 解析漏洞、文件头伪造 (图片马)。

**3. 伪造与实体漏洞**
*   **SSRF (服务端请求伪造)：** `dict://`, `gopher://` 协议深度利用，DNS 重绑定。
*   **XXE (XML 外部实体注入)：** 读取本地文件、Blind XXE (利用 OOB 带外数据)。

### 四、 高级特性与代码审计
**1. 语言特性与缺陷**
*   **PHP：** 弱类型比较 (`==` 哈希碰撞)、内置函数缺陷 (`preg_match` 回溯)、变量覆盖 (`$$`)。
*   **Python：** 沙箱逃逸 (`__builtins__`, `__class__`)。
*   **Java：** 内存马 (MemShell) 技术、JNDI 注入。

**2. 反序列化与模板注入**
*   **PHP 反序列化：** 魔术方法触发 (`__wakeup`, `__destruct`)、Phar 反序列化。
*   **SSTI (模板注入)：** 利用模板引擎的上下文执行任意代码。

### 五、 Web 密码学与安全架构
*   **密码学缺陷：** 哈希长度扩展攻击 (Hash Length Extension)、Padding Oracle Attack、弱伪随机数生成器 (PRNG)。
*   **WAF 绕过体系：** 垃圾数据填充、分块传输绕过 (Chunked Transfer)、编码混淆。
*   **现代云架构：** HTTP 请求走私 (HTTP Request Smuggling)、云服务 Metadata SSRF 窃取凭证。
