## 🎯 终极目标：我们在 Web 题里到底在找什么？

很多初学者学了 SQL 注入、XSS，但在真正的 CTF 题目面前依然一头雾水。在开始学习之前，你必须明确我们的最终目标是什么：

1.  **获取 Flag：** CTF 的精髓在于夺旗。Web 题的 Flag 通常藏在服务器的某个文件里（如 `/flag`），或者数据库的某个表里，甚至需要你拿到服务器的最高权限。
2.  **实现 RCE (Remote Code Execution)：** 远程代码执行是 Web 狗的终极浪漫。无论是通过文件上传传木马，还是通过反序列化触发漏洞，我们 90% 的努力都是为了能在服务器上执行一句 `cat /flag`。
3.  **GetShell 与权限维持：** 利用漏洞向服务器植入后门（Webshell），通过工具（如蚁剑、冰蝎）连接，接管目标机器。

## 🧭 上篇：CTF Web 学习路线与避坑指南

很多新手容易犯的第一个错误，就是连 HTTP 协议都没搞懂，就直接去背 SQL 注入的 Payload。基础不牢，地动山摇。

### 第一阶段：筑基期（绝对不要跳过！）
*   **🌐 网络基础（重中之重）：** 深刻理解 HTTP/HTTPS 协议。请求头 (Headers)、响应头、Cookie、Session、状态码 (200, 302, 403, 500 等)。
*   **🖥️ 前端基础：** HTML/CSS（看懂表单提交），JavaScript（DOM 操作，XSS 的基础），浏览器安全策略（同源策略 SOP、跨域 CORS）。
*   **⚙️ 后端与底层环境：** Linux 基础命令（文件操作、权限、管道符）；MySQL 基础语法；至少掌握一门后端语言（首推 PHP，其次 Python），能看懂代码逻辑即可。

### 第二阶段：核心漏洞图谱（经典漏洞实战）
掌握 Web 安全中最经典的漏洞模型。不仅要知道"怎么黑"，更要知道"为什么能黑"（成因）以及"怎么修"（防御）。
*   **注入类漏洞：** SQL 注入、命令注入 (RCE)。
*   **文件操作类：** 文件上传（绕过黑白名单）、文件包含 (LFI/RFI)。
*   **客户端漏洞：** XSS (跨站脚本攻击)、CSRF。
*   **逻辑漏洞：** 越权访问、支付逻辑漏洞、验证码绕过。

### 第三阶段：进阶与现代 Web 安全（拉开差距的考点）
*   **反序列化漏洞：** PHP 反序列化 (POP 链)、Java 反序列化 (CC 链)、Python Pickle。
*   **服务端请求伪造：** SSRF (内网探测、打 Redis/MySQL)。
*   **模板注入 (SSTI)：** Jinja2、Twig 等引擎的沙箱逃逸。
*   **现代特性：** Node.js 原型链污染、JWT 伪造。

### 🛑 防走弯路指南（避坑必看）
1.  **🚫 戒掉对自动化工具的依赖：** 新手期绝对不要一开始就用 Sqlmap 或 AWVS。如果连手动注入的闭合都没搞懂，扫描器只会让你知其然而不知其所以然。
2.  **🔍 看重源码，学会本地调试：** 遇到黑盒题目做不出，试着在本地写一段类似的 PHP 代码，`var_dump()` 打印一下变量，看看底层到底发生了什么。
3.  **📖 赛后复盘重于比赛本身：** 比赛时做不出很正常。比赛结束后，一定要去看别人的 Writeup！学习大佬的切入点和 Bypass 思路。
4.  **🧱 抓核心，轻偏门：** 很多 CTF 题目的 Trick 非常偏门，不要花太多时间死记硬背偏门函数，优先掌握底层的核心机制。

## 🗺️ 中篇：CTF Web 核心知识体系字典

本字典旨在对 Web 安全领域的关键技术进行系统性分类，作为日常学习、做题与代码审计的速查索引。

### 一、 信息搜集与敏感泄露 (Information Gathering)
在 CTF 中，找不到源码往往连题目的门都进不去。
*   **源码泄露：** `.git` 泄露、`.svn` 泄露、`.DS_Store` 泄露。
*   **备份文件：** 常见的备份后缀如 `.bak`, `.zip`, `.tar.gz`, `~`。
*   **探针与配置泄露：** `phpinfo()` 页面、`robots.txt`、Spring Boot 接口泄露。

### 二、 协议与网络基础层
*   **HTTP 请求结构：** Method, URI, Version, Headers (Host, User-Agent, X-Forwarded-For)。
*   **状态保持机制：** Cookie, Session, JWT 签名机制。
*   **同源策略与跨域：** SOP 限制范围, CORS 配置不当, JSONP 劫持。

### 三、 核心漏洞图谱 (服务端)
**1. 注入类漏洞 (Injection)**
*   **SQL 注入：** 联合查询、报错注入、布尔盲注、时间盲注、堆叠查询。（*进阶：绕过空格/逗号、无列名注入*）
*   **命令注入 (RCE)：** 管道符利用 (`|`, `&`, `;`)、无回显 RCE。

**2. 文件操作漏洞**
*   **文件包含 (LFI / RFI)：** 伪协议利用 (`php://filter`, `data://`)、日志包含。
*   **文件上传 (File Upload)：** MIME 绕过、后缀绕过、解析漏洞、图片马。

**3. 伪造与实体漏洞**
*   **SSRF (服务端请求伪造)：** `dict://`, `gopher://` 协议深度利用，DNS 重绑定。
*   **XXE (XML 外部实体注入)：** 读取本地文件、Blind XXE。

### 四、 高级特性与代码审计
*   **语言特性缺陷：** PHP 弱类型比较 (`==` 哈希碰撞)、正则回溯、变量覆盖。
*   **反序列化：** PHP 魔术方法触发 (`__wakeup`, `__destruct`)、Phar 反序列化。
*   **SSTI (模板注入)：** 利用模板引擎 (Jinja2/Twig) 的上下文执行任意代码。

### 五、 Web 密码学与防御绕过
*   **密码学缺陷：** 哈希长度扩展攻击 (Hash Length Extension)、伪随机数预测。
*   **WAF 绕过体系：** 垃圾数据填充、分块传输绕过、编码混淆。

## ⚔️ 下篇：实战兵器库与优质资源

纸上得来终觉浅，绝知此事要躬行。理论看完了，该去哪里练手呢？

### 🔧 必备兵器谱 (Tools)
*   **Burp Suite：** Web 选手的灵魂，一切抓包、修改、重放、爆破的基石。
*   **Python + Requests：** 当你需要进行时间盲注或者复杂的爆破时，自己写脚本是唯一的出路。
*   **Webshell 管理器：** 蚁剑 (AntSword)、冰蝎 (Behinder)、哥斯拉 (Godzilla)。
*   **常用扫描辅助：** `dirsearch` (目录爆破), `HackBar` (浏览器辅助插件)。

### 🎯 优质靶场推荐 (Playgrounds)
*   **PortSwigger Web Security Academy：** (强烈推荐) Burp 官方出的免费靶场，分类极其清晰，史上最强 Web 安全入门教材。
*   **BUUCTF：** 国内题目质量极高的平台，收录了大量历届大型 CTF 比赛的真题。
*   **CTFHub：** 知识点分类细致，非常适合用来针对性地巩固某个特定漏洞。

### 📚 信息获取源 (Resources)
*   **CTF Wiki：** (`ctf-wiki.org`) CTF 界的百科全书，遇到不懂的名词第一反应应该是去这里查。
*   **Seebug / FreeBuf / 先知社区：** 追踪最新漏洞情报和高质量技术分析文章的绝佳去处。
